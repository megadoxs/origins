Origins Mod

Origins is a minecraft mods that gives every player the opportunity to choose his origin (similar to a race) and have powers that reflect that origin. If this was all the Origins mod had it wouldn't have been so great since as you will see, there is only 6 origins by default. You may or may not understand now that I told you that there is 9 orgins by default, that you can have more. As I will show you later, this mods not only adds hard codded orgins, but also give the opportunity to make your own.

Default Origins

If you use the origin mod by default, you only have access to the nine following origins:

  • Enderian
  • Elytrian
  • Arachnid
  • Merling
  • Blazeborn
  • Shulk
  • Phantom
  • Avian
  • Feline

Personalised Origins

Personalised Origins or custom Origins are origins that are not by default in the mod and can be added using a modified version of minecraft vanilla datapack format. Once the origins have been added in the datapack folder, they can be selected the same way you would select any of the default one. A bunch of origins made by third parties and help to make them can be found on Origins discord server.

Defining a Powers

Powers are used to give functionality to origins, they are json file that goes inside the power folder of your origin datapack.

Generaly they can have up to 8 arguments from this list:

  • Type
  • Actions
  • Condition
  • Name
  • Description
  • Hidden
  • Badge
  • Loading Priority

*Some power type don't accept action or only specificts type of action, same goes for condition.

To make all this more clear, Next to this text is a full example of the coded of a power that would make you burn if you are exposed to the sun. Then, underneath, you can see each part of the code individually.

Lorem ipsum dolor sit amet.

{
    "type":"origins:action_over_time",
    "entity_action":{
        "type": "origins:set_on_fire",
        "duration": 5
    },
    "condition":{
        "type": "origins:exposed_to_sun"
    },
    "interval": 200,
    "name":"Sun Burn",
    "description":"burns you when you stand under the sun",
    "hidden": false
     }

Power Types

power types are in a way a countainer that you can only be access if the general condition for the power type happend. Might be confusing so here is an example: You want to do something every 10 seconds, the best way to do so would be to use a action over time that does wathever is inside this "power type" every specified amount of time. This power type would then execute Actions that you have put in it if the Conditions that you gave the "power type" to execute are true.

Lorem ipsum dolor sit amet.

{
    "type":"origins:action_over_time",
    "entity_action":{
        "type": "origins:set_on_fire",
        "duration": 5
    },
    "interval": 200
}

Actions

actions are events that you want to happend when the power types executes, which they are part of. e.g. you want to brun the player every 10seconds. you would use an action over time power type make it so that it executes every 10seconds and use a burn entity action.

Conditions

Conditions are well conditions that you want to be true before the power types executes, which they are also part of. e.g. you want to brun the player every 10seconds if the player is exposed to sun. you would use an action over time power type make it so that it executes every 10seconds and use a burn entity action and an exposed to sun entity condition.
